# Device Foundation

The design system targets source-defined device classes for phone-first, tablet, and desktop layouts.

To build fluid, hybrid, PWA-ready applications, the system separates **responsive width** from **physical hardware capabilities**:

---

## 1. Viewport Class (Responsive Layouts)

The viewport class reflects the **current layout width** only. It determines how much horizontal room is available and mirrors the CSS breakpoints.

## 2. Device Type (Hardware & Capabilities)

The device type tracks **actual hardware capabilities**, irrespective of orientation. For example, a real iPad rotated horizontally is 1366px wide, but its input and behavioral profile is still touch-first. 

We classify hardware statically at startup to enable high-fidelity touch interactions, notch-aware margins, and Apple Pay integrations:

- **Touch Interaction:** coarseness of pointer, touch-start hooks (`isTouchDevice`)
- **iOS/iPadOS User-Agent detection:** disambiguating iPadOS Safari (which requests Mac desktops) via touch-point thresholds.
