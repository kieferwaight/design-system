# Color

Color is built in layers. Each layer composes the layer below it, so design
tokens, semantic roles, palettes, themes, and generative systems can evolve
independently — change one without rewriting the rest.

<Canvas of={ColorStories.DemoLayerDiagram} meta={ColorStories} />

A single **Color Token** becomes a **Swatch**, swatches group into a **Palette**,
palettes assemble into a **Base Color System**, and a **Generative System** turns
a handful of seeds into all of it at once.

---

## The Apple Ecosystem

The design system adopts [Apple's color
architecture](https://developer.apple.com/design/human-interface-guidelines/color)
as its foundation. Apple provides a complete, battle-tested visual language —
system colors, system grays, dynamic labels, backgrounds, fills, separators,
materials, and accessibility variants. These primitives are the bedrock that
semantic tokens and generated themes are built upon.

### Core Colors

The neutral foundation: Apple's six system grays, from the lightest surface
(Gray 6) through to the strongest (Gray 1).

### Accent Colors

Twelve harmonious system hues form the expressive layer of the palette.

## OKLCH Generative Color System

The Apple primitives give a fixed color language. The **OKLCH Generative Color
System** extends that foundation by creating entire color families from a small
number of inputs. Instead of defining hundreds of colors by hand, a few seed
values generate complete, perceptually balanced scales.

### What Is Generated?

Two primary families come out of the engine.

**Neutral Scale** — surfaces, elevation, borders, typography, shadows, and the
structural hierarchy of the UI.

<Canvas of={ColorStories.DemoNeutralRamp} meta={ColorStories} />

**Accent Scale** — interactive controls, brand color, focus states, charts,
data visualization, and status.

<Canvas of={ColorStories.DemoAccentRamp} meta={ColorStories} />

## Chroma

Chroma controls color intensity. Higher chroma reads stronger; lower chroma
reads softer. The same layout, same hue — only the chroma changes.

<Canvas of={ColorStories.DemoChromaVariants} meta={ColorStories} />

### Why It Matters

Different products want different personalities. Enterprise software favors
restrained chroma; consumer products lean stronger; brand experiences can push
it all the way to expressive. One seed covers the whole range.

---

## Hue

Hue determines the color family. The accent scale below is generated by

### Perceptual Color Behavior

The Neutral Scale is not linear — it is tuned to how humans actually perceive
color.

**Chroma distribution.** A touch of chroma peaks near the middle of the scale
and falls toward both ends, so mid-tones feel alive while the extremes stay
clean.

<Canvas of={ColorStories.DemoNeutralChromaCurve} meta={ColorStories} />

**Hue drift.** Subtle hue shifts across the ramp create depth. The interface
feels more natural while still reading as neutral.

<Canvas of={ColorStories.DemoHueDriftCompare} meta={ColorStories} />

## Accent Scale

The Accent Scale generates interaction colors.

<Canvas of={ColorStories.DemoAccentSwatches} meta={ColorStories} />

### Perceptual Saturation Curve

The scale intentionally grows richer before it grows darker — chroma follows a
bell curve, matching the expectation that color deepens before it dims.

<Canvas of={ColorStories.DemoAccentChromaCurve} meta={ColorStories} />

### Hue Drift

Accent colors shift subtly as they darken, moving from a cyan-leaning blue in
the light end toward an indigo-leaning blue in the dark end. The family gains
depth while staying recognizably one color.

<Canvas of={ColorStories.DemoAccentHueDrift} meta={ColorStories} />

## Theme Generation

A complete theme can be generated from a small number of inputs. Identical UI —
only the generated color seeds differ.

<Canvas of={ColorStories.DemoThemeGeneration} meta={ColorStories} />

A product can support brand themes, customer themes, white-label deployments,
and per-user personalization — all without rewriting components.
